Ubuntu Server + nginx + gunicorn + PostgreSQL


STEP 1 - Update the server and install dependencies
----------------------------------------------------
Connect to server via SSH, then run:
(installs Python 3, pip, git, nginx and PostgreSQL)

sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip python3-venv git nginx postgresql postgresql-contrib
This installs Python 3, pip, git, nginx and PostgreSQL in one command.


STEP 2 - Configure PostgreSQL
------------------------------
sudo -u postgres psql

Then inside the PostgreSQL consoler:

CREATE DATABASE gesm_leave;
CREATE USER gesm_user WITH PASSWORD 'choose_a_strong_password';
GRANT ALL PRIVILEGES ON DATABASE gesm_leave TO gesm_user;
\q

IMPORTANT: Write down the password needed it in Step 5 (DB_PASSWORD).

STEP 8 - Configure gunicorn as a systemd service
--------------------------------------------------
Create the service file:

sudo nano /etc/systemd/system/gunicorn.service

Pastet:

[Unit]
Description=GESM Leave Management gunicorn daemon
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/var/www/gesm-leave
EnvironmentFile=/var/www/gesm-leave/.env
ExecStart=/var/www/gesm-leave/env/bin/gunicorn \
          --workers 3 \
          --bind unix:/run/gunicorn.sock \
          hrapp.wsgi:application

[Install]
WantedBy=multi-user.target

Save and run:

sudo systemctl daemon-reload
sudo systemctl start gunicorn
sudo systemctl enable gunicorn
sudo systemctl status gunicorn

"active (running)" in green 


STEP 9 - Configure nginx
--------------------------
the nginx config file:

sudo nano /etc/nginx/sites-available/gesm-leave

Paste :

server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    location = /favicon.ico { access_log off; log_not_found off; }

    location /static/ {
        root /var/www/gesm-leave;
    }

    location /media/ {
        root /var/www/gesm-leave;
    }

    location / {
        include proxy_params;
        proxy_pass http://unix:/run/gunicorn.sock;
    }
}

Then run:

sudo ln -s /etc/nginx/sites-available/gesm-leave /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
sudo ufw allow 'Nginx Full'

COMMON PROBLEMS AND SOLUTIONS (FROM FORUM ONLINE) 
------------------------------

502 Bad Gateway
-> gunicorn is not running
-> sudo systemctl restart gunicorn

Static files (CSS) not loading
-> python manage.py collectstatic --noinput
-> sudo systemctl restart nginx

Permission denied on socket
-> sudo chown www-data:www-data /run/gunicorn.sock

Migration error
-> python manage.py showmigrations
-> python manage.py migrate --run-syncdb

Emails not sending
-> Check EMAIL_HOST and EMAIL_HOST_PASSWORD in .env

ALLOWED_HOSTS error
-> Add your domain or IP to ALLOWED_HOSTS in .env

500 Internal Server Error
-> sudo journalctl -u gunicorn -f to see the exact error
